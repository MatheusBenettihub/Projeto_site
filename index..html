Artigo 001 - O que é uma branch?

Uma branch é uma linha de desenvolvimento separada dentro de um repositório Git. Ela permite que você trabalhe em funcionalidades diferentes sem afetar o código principal.

Trabalhando com Merges

Artigo 002 - O que é um merge?

Merge é o processo de unir duas branches. Isso permite que mudanças feitas em uma branch sejam integradas à branch principal ou a outra qualquer.

Conflitos no Git

Artigo 003 - E quando há conflito?

Conflitos acontecem quando duas branches alteram a mesma linha de um arquivo. O Git não sabe qual manter, então você precisa resolver manualmente.
